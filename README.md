资治通鉴_让世界看到中国千年文化
以外交传播视角面向全球展示中国文化的 24 小时自动化内容生产与知识呈现引擎


产品定位	以《资治通鉴》为起点，将中国古典历史文本转化为可读、可学、可传播、可持续迭代的中英双语文化内容产品。
项目缘起	项目源于一位拥有 30 余年外交工作经历的资深外交官长期积累与心血投入，希望以更容易被国际读者理解和接受的方式，系统化展示中国文化、中国历史与中国叙事。
核心引擎	以 OpenClaw 为运行与编排底座，结合 detection、annotation、index、style、format、test 等专用 subagent，实现 24 小时自动完善。

一句话概括
这不是一个通用问答式古籍助手，而是一套以 OpenClaw 为核心运行时、面向《资治通鉴》全文双语翻译与高质量注释的专用文化内容生产系统。

一、项目使命与产品愿景
这个项目首先不是一个单纯由技术驱动的实验，而是一项带有明确文化使命的长期工程。它源于一位拥有 30 余年外交工作经历的资深外交官多年的观察、积累与心血：在长期国际交流中，他深切感受到，世界对中国文化始终有兴趣，但真正能被国际社会顺畅理解、持续阅读、系统传播的高质量内容仍然稀缺。
因此，我们的核心目标，不只是把《资治通鉴》做成一个可搜索、可阅读的数字文本，而是把它做成一个面向全球传播中国文化的高质量内容产品。通过中英双语翻译、系统化注释与统一的表达风格，让更广泛的国际读者真正读懂中国古代历史叙事、制度设计、政治智慧与文明精神。
从这个背景出发，这个项目的价值不在于“多快地生成一段答案”，而在于“能否长期、稳定、低成本地生产出有出版品质、有统一 taste、有历史文化厚度的内容版本”。《资治通鉴》是起点，但这套能力最终可以扩展到中国传统文化的其他经典文本、专题课程、视频脚本、海外传播素材与教育产品。
产品精神
以高标准内容输出中国文化，以国际传播视角讲述中国文化；把一位资深外交工作者长期积累的判断、标准与文化使命，沉淀为可以规模化复制的产品能力。





二、为什么是《资治通鉴》
《资治通鉴》兼具历史叙事深度、制度信息密度与文明解释价值，是极具代表性的中国文化内容载体。它既能承载中国古代政治、军事、制度与人物史，也适合衍生为阅读产品、知识服务、课程内容与国际传播素材。
但《资治通鉴》的阅读门槛极高：人物、地名、官名、度量衡、谚语和典故密集出现，缺乏注释时，现代中文读者已经不易通读，国际读者更难建立连续理解。传统人工方式虽然质量高，但速度慢、成本高、难以形成全书级别的一致性。
这正是本产品切入的机会：在尊重出版级质量标准的前提下，把“全文翻译 + 全书注释 + 风格统一 + 自动迭代”做成一个持续运转的内容生产系统。

内容形态	可做成网页、App、电子书、知识专题、课程资料、视频脚本与对外传播素材。

五、核心产品能力
1.中英双语译注一体化：原文、今译、英译与中英双语注释同屏联动，提升阅读与传播效率。
2.六类重点知识自动标注：对人名、地名、官名、度量衡、谚语、典故进行检索与注解，构成核心注释体系。
3.注释量可控：按照“标准高的注释多 > 标准高的注释少 > 标准低的注释多”的原则，优先保证质量，再控制密度。
4.全文风格统一：基于已出版、已校对版本沉淀出的 taste，确保整部作品的注释语气、深度、长度与解释方式一致。
5.跨章节去重与复用：通过 index subagent 与单书知识索引，避免重复加注、重复翻译与风格漂移。
6.24 小时自动完善：系统可持续运行，按章节、按片段、按任务队列不断产出、校正与回写记忆。





六、OpenClaw 为什么是核心
在这个项目里，OpenClaw 不是一个可有可无的外壳，而是整个系统的运行时与编排中心。它负责会话、路由、调度、节点承载以及整体任务流管理，是所有自动化能力落地的底座。
更准确地说，OpenClaw 解决的是“让这套复杂流水线可靠运转”的问题：一段文本进入系统后，需要依次经过发现、生成、检索、统一、格式校正、测试与回写等多个步骤。没有一个强约束的 orchestrator，就很难在全书级项目中保持连续性、一致性和可运维性。
维度	              内容
会话与任务入口	每一章、每一段正文都可作为任务流进入 OpenClaw，形成可追踪、可恢复的处理链路。
路由与编排	把同一块文本依次路由给 detection、annotation、index、style、format、test 等 subagent。
节点承载	OCR、grep、RAG 检索、QA runner、LiteLLM 路由等外围能力，都以工具或服务方式挂在 OpenClaw 流程中。
控制面与监控	适合后续建设批处理监控、人工复核、日志追踪与多批次任务调度能力。




七、产品核心骨架：六层架构

八、Subagent 设计：每一层具体做什么
Subagent	核心任务	详细制作思路
Detection	找到该加注的地方	先用 grep/规则/词表做高召回扫描，再用模型补充隐性典故与制度词，输出候选 span、类型、优先级与理由。
Annotation	生成注释	针对人名、地名、官名、度量衡、谚语、典故分别用模板生成中英双语短注/长注，输出结构化结果。
Index	防止重复加注	结合精确索引与单书 RAG，判断是否已注、是否应轻注、是否为同名异人，并推荐复用历史注释。
Style	统一全文 taste	以已出版版本为正样本，以机器腔、百科腔、论文腔为负样本，只改表达，不改事实。
Format	格式校正	统一标点、字段、编号、Markdown/JSON 结构与 span 对齐，尽量规则化，减少模型浪费。
Test	回归测试	检测漏注率、重复率、风格偏差、格式合法性与回归样本通过率，为迭代提供量化反馈。

这个设计的关键不是让每个 agent 都“大而全”，而是让每个 subagent 只承担一个职责，形成稳定、可替换、可优化的流水线。这种扁平化结构也更适合长期工程迭代。




九、如何做到“最省 token 的方式自动化加高质量注释”
7.先规则后模型：先用 grep、词表、正则与实体库做第一轮高召回筛查，把昂贵模型留给真正需要解释的部分。
8.先检索后生成：已有注释优先复用，只有在历史注释完全不适配时，才重新生成或重写。
9.分块处理、按需拼接：正文统一转为 Markdown 后切块处理，只在需要跨段理解时调用长上下文模型。
10.不同子任务用不同模型：扫描、格式、测试可用经济模型，关键生成和风格统一再调用更强模型。
11.结果回写记忆：每次新注释、风格修订和错误案例都会进入记忆层，下一轮任务直接收益。


一句工程原则
让模型负责判断与表达，让规则和检索负责确定性与复用；把 token 花在最有价值的地方。

十一、质量标准与 taste 体系
对于这类文化产品来说，真正决定上限的不是“是否能生成”，而是“生成出来是否像一个已经出版、已经校对过一年多的版本”。因此，taste 不是锦上添花，而是质量系统的核心部分。
维度	内容
事实标准	注释必须准确解释人物、地名、制度、典故与语义关系，不凭空扩写。
密度标准	该注则注，不该注不注；首次详注、近距轻注、重复跳注。
表达标准	语言简洁、稳定、可读，避免百科腔、论文腔、翻译腔和模型腔。
双语标准	重要注释尽量保持中英成对，方便国际传播与跨语种阅读。
全书一致性	不同章节、不同批次、不同模型产出的内容，最终都要回到同一套表达标准上。

我们追求的不是“看起来像 AI 很聪明”，而是“看起来像一套稳定成熟的文化出版产品”。

16.不是只服务一本书，而是先以《资治通鉴》证明可行，再扩展到更多传统文化经典与国际传播产品。




结语
这个项目的意义，不只是为《资治通鉴》做一套工具，而是验证一种新的文化生产方式：把中国传统文化中最有价值的内容，转化为面向全球读者的现代产品形态；把本来高度依赖人工经验的出版级工作流，沉淀为可以规模化复制的智能系统。
从这个意义上说，OpenClaw 是系统的底座，subagent 是能力的骨架，风格与记忆是内容的灵魂，而《资治通鉴》则是最能证明这套方法论的第一块里程碑。
